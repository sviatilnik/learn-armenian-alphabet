# Техническое задание: CLI-игра "Изучаем армянский алфавит" (Go)

## 1. Общее описание
Разработать консольное приложение (CLI) на языке Go для интерактивного изучения армянского алфавита. Приложение должно работать в двух режимах, вести статистику ответов пользователя и предоставлять обратную связь.

## 2. Функциональные требования

### 2.1. Запуск и главное меню
При запуске приложения пользователь видит приветствие и меню выбора режима:
```
Добро пожаловать в тренажёр армянского алфавита!
Выберите режим:
1. Назвать букву (по символу)
2. Назвать звук/транслитерацию (по символу)
3. Выход
Введите номер режима:
```
*   Если пользователь выбирает `3`, программа завершает работу.
*   Если пользователь вводит несуществующий номер или строку, программа выводит сообщение об ошибке и повторяет запрос ввода.

### 2.2. Режимы игры

#### Режим 1: «Назвать букву»
*   **Суть:** Пользователю показывается символ армянской буквы (в случайном регистре). Задача пользователя — ввести название этой буквы (транслитерацию или русское/английское название, согласно заранее заданному словарю).
*   **Пример:** На экране выводится `Ա` или `ա`. Ожидаемый ответ: `a` или `айб` (зависит от выбранного словаря).

#### Режим 2: «Назвать звук»
*   **Суть:** Пользователю показывается символ армянской буквы. Задача пользователя — ввести звук, который она дает (транслитерацию кириллицей или латиницей), или ее примерное звучание.
*   **Пример:** На экране выводится `Բ` или `բ`. Ожидаемый ответ: `b`.

### 2.3. Игровой процесс
1.  После выбора режима игра начинается.
2.  Программа генерирует случайную букву из алфавита.
3.  Случайным образом выбирается регистр буквы (заглавная или строчная) для отображения. Вероятность должна быть ~50/50.
4.  На экран выводится приглашение:
    ```
    Буква: [Буква_в_случайном_регистре]
    Ваш ответ:
    ```
5.  Пользователь вводит свой вариант и нажимает Enter.
6.  Программа считывает ввод, сравнивает его с правильным ответом для данного режима и буквы.
7.  Сразу после ввода выводится результат текущего хода:
    *   Если ответ верный: `Правильно! 👍`
    *   Если ответ неверный: `Неправильно. Правильный ответ: [Ожидаемый_ответ]`
8.  После этого автоматически (без запроса на продолжение) генерируется следующий вопрос. Игра должна быть бесконечным циклом до тех пор, пока пользователь не прервет ее вручную (Ctrl+C).

### 2.4. Сбор статистики
В процессе игры для каждого хода необходимо собирать и выводить статистику.
**Формат вывода статистики на экран перед каждым новым вопросом:**

```
--------------------------------------------------
Ход #1 | Буква: Ա (ожидание: a) | Ваш ответ: a | Верно
Ход #2 | Буква: բ (ожидание: b) | Ваш ответ: b | Верно
Ход #3 | Буква: Մ (ожидание: m) | Ваш ответ: n | Неверно
--------------------------------------------------
```
*   Статистика накапливается и выводится перед каждым следующим ходом, показывая историю последних N действий (опционально можно показывать всю историю сессии).

## 3. Структуры данных

### 3.1. Словарь алфавита
Необходимо создать структуру или мапу для хранения данных об алфавите. Ключом является символ Unicode (руна) армянской буквы в нижнем регистре (для однозначности), а значением — структура с вариантами ответов.

```go
type ArmenianLetter struct {
    Uppercase     rune   // Заглавная форма (Ա, Բ, Գ...)
    Lowercase     rune   // Строчная форма (ա, բ, գ...)
    NameAnswer    string // Ожидаемый ответ для режима "Назвать букву" (напр. "айб", "бен", "гим")
    SoundAnswer   string // Ожидаемый ответ для режима "Назвать звук" (напр. "a", "b", "g")
}
```
**Важно:** Наполнить данными минимум 5-7 букв для теста, но структура данных должна легко позволять добавить все 39 букв алфавита.

### 3.2. Структура статистики
Для хранения информации о каждом ходе:
```go
type GameStat struct {
    TurnNumber     int
    DisplayedRune  rune   // Какая буква была показана пользователю
    ExpectedAnswer string // Какой ответ ожидался
    UserAnswer     string // Что ввел пользователь
    IsCorrect      bool   // Результат
}
```

## 4. Требования к вводу/выводу

*   **Кодировка:** Консоль должна корректно отображать Unicode (армянские символы). На Unix-подобных системах (Linux/macOS) это обычно работает из коробки. На Windows 10+ рекомендуется использовать шрифт, поддерживающий Unicode (например, Cascadia Code, Consolas) и терминал вроде Windows Terminal.
*   **Регистр ответов:** Сравнение ответов должно быть **регистронезависимым** (т.е. "B" и "b" считать одинаковыми).
*   **Пробелы:** Лишние пробелы в начале и конце ввода должны игнорироваться (обрезка строки).

## 5. Технические детали реализации (Go)

*   **Пакеты:** Использовать только стандартную библиотеку Go.
    *   `fmt`, `bufio`, `os` для ввода/вывода.
    *   `math/rand` для генерации случайных букв и регистра.
    *   `time` для инициализации генератора случайных чисел.
    *   `strings` для обработки строк (ToLower, TrimSpace).
    *   `unicode` для определения/работы с регистром (опционально).
*   **Точка входа:** Функция `main()`.
*   **Основной цикл:** Бесконечный цикл `for { ... }` с условием выхода по команде (режим 3) или по прерыванию (Ctrl+C).
*   **Обработка прерывания (Ctrl+C):** Желательно добавить обработку сигнала `os.Interrupt` (SIGINT), чтобы при нажатии Ctrl+C программа не завершалась аварийно, а выводила итоговую статистику и прощалась.

## 7. Критерии готовности

1.  Приложение компилируется без ошибок.
2.  Реализовано меню с выбором режима и выходом.
3.  Корректно отображаются армянские буквы в разных регистрах.
4.  Реализована логика проверки ответов для обоих режимов.
5.  Ведется и выводится статистика в требуемом формате.
6.  Код хорошо структурирован, разбит на функции (например, `printMenu`, `getRandomLetter`, `checkAnswer`, `printStatistics`).
